# Incident Response Playbook

## 1. Identificazione
- Rilevare l'incidente tramite alert, audit log o segnalazioni.
- Classificare severità e impatto.

## 2. Contenimento
- Isolare i sistemi coinvolti.
- Bloccare credenziali/API key compromesse.

## 3. Eradicazione
- Rimuovere la causa root (vulnerabilità, configurazioni errate).
- Applicare patch o mitigazioni.

## 4. Recovery
- Ripristinare servizi e verificare integrità.
- Monitorare per recidive.

## 5. Post-Incident
- Report dettagliato con timeline.
- Azioni correttive e preventive.
- Aggiornamento playbook e formazione.
